## EXP-009 · OpenProse 与轻量 contract 试点

**日期**：2026-04-29  
**来源**：INBOX（cursor + claude-desktop）  

**结论**：完整 OpenProse VM 成本高，不适合默认路径。高价值调研与复杂 review 可在任务 description 中附加轻量 **Requires / Ensures / Strategies**（及完成标准），无需改 MCP；其它任务不强制。连续多轮验证收益后再考虑固化进规则或模板。  

**应用**：Orchestrator 对 critical research、复杂 review 发单时附带 contract 块；执行方按字段交付，避免范围漂移。轻量模式控制 token 增量在约 10%–30%。  

**RuleStore 方向**：任务类型或 `priority`+`type` 组合触发「description 含 contract 区块」的检查项（人工或半自动）。
