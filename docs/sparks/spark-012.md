## SPARK-012 · 枚举值视觉锚点——抢占用户认知习惯

**日期**：2026-04-25
**来源**：WebUI 体验复盘，priority/status 颜色区分度弱，下拉框选项无视觉标识

**洞见**：
枚举值（status、priority、type 等）应当**始终与固定的颜色或图标绑定**，
而不仅仅在展示时附色——下拉选项、表单标签、列表行、详情页，任何出现枚举值的地方，
颜色/图标必须同步出现。

目标是让用户反复接触同一个"信号→含义"映射，直到无需阅读文字就能识别状态。
这种认知一旦内化，就成为产品的隐性护城河——用户迁移到其他工具时，
需要重新建立认知习惯，迁移成本显著上升。

**从细节到壁垒的路径**：
1. 设计阶段：为每个枚举值分配不可复用的颜色+图标（避免 running/blocked 撞色）
2. 实现阶段：枚举的视觉规格抽成 token/constant，一处定义，全局引用
3. 坚守阶段：任何新功能引入枚举值时，必须同步配色——通过 code review 强制执行

**产品延伸**：
- 同理适用于 Agent 状态（online/offline/busy）、任务类型（code/review/test）
- 配色系统一旦稳定，可以输出为"设计规范"，成为品牌资产
