# SPARK-029 · 确定性归代码，非确定性归 Agent——系统性边界 review

**成熟度**：🟡 近期可做

## 洞见

在 nerve-hub 的实际运行中发现一类隐患：某些本应由代码保证的确定性行为（唯一 ID 生成、序号递增、状态机跃迁规则），被交给了 Agent 去推断或生成。Agent 是概率性系统，做确定性工作天然不可靠。

具体触发点：Sprint 编号、任务序号等 ID 被 Agent "猜"出来，容易产生重复或不连续，排查成本高。

## 核心原则

> 确定性任务归编程语言，非确定性任务归 Agent。

- **代码负责**：唯一ID、序号、状态机转换、约束校验、幂等逻辑
- **Agent负责**：自然语言理解、意图推断、内容生成、模糊决策

## 行动方向

对整个 nerve-hub 工程做一次系统性 review，识别哪些地方当前在用 Agent 做本该由代码保证的事，形成重构清单，分批修复。

重点扫描区域：
- 任务 ID / Sprint 编号生成逻辑
- 状态流转规则（pending → running → done 边界是否有代码 guard）
- Agent 传入的字段是否有服务端校验（不能只靠 Agent "自觉"）
