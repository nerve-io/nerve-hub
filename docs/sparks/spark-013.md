## SPARK-013 · 减少用户操作——有源数据改 pick，禁 input

**日期**：2026-04-25
**来源**：WebUI 复盘，Assignee 自由输入、Dependencies 手动填 ID，体验差

**洞见**：
**凡是有明确数据来源的字段，都不应该让用户手动输入。**
数据已经在系统里，让用户去记 ID、拼字符串，是把系统的认知负担转嫁给了人。

具体场景：
- `Assignee`：已有 Agents API，改为从接口动态拉取，选而非填
- `Dependencies`：已有 Tasks API，改为多选控件，显示标题而非 ID
- `Project`：新建 Task 时，project 列表应当下拉选，而非填 ID
- 未来凡引入新的关联字段，优先考虑 pick 控件

**设计原则**：
> "最好的交互，是用户意识不到自己在交互。"

**产品延伸**：
- 结合 SPARK-012，pick 控件的选项也应带枚举颜色/图标
- 动态数据的加载状态（loading/empty/error）需要统一设计，避免白屏或闪烁
