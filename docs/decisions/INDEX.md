# Decisions — 架构与产品决策记录

> 轻量版 ADR（Architecture Decision Record），记录重要决策的背景、选项、理由和影响。
> 不是设计文档，不是技术方案，是对"为什么这样做"的追溯。

## 决策记录模板

新决策按 `dec-<NNN>.md` 命名，内容遵循以下模板：

```markdown
# DEC-<NNN>: <决策标题>

**日期**: YYYY-MM-DD
**状态**: 提议 / 已采纳 / 已废弃
**决策者**: Neil

## 背景
需要做什么决定，为什么现在做

## 决策
我们决定做什么

## 考虑过的替代方案
- 方案 A：简述 + 为什么没选
- 方案 B：简述 + 为什么没选

## 影响
- 正面：带来什么好处
- 负面：付出什么代价
- 后续：需要跟进什么

## 相关
- 相关 SPARK / EXP 编号
- 相关代码文件
```
