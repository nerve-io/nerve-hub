# Deprecated — 废弃特性归档

> 不再维护或已移除的特性记录，方便未来捡回时快速定位上下文。

## 用途

- 记录已下线的特性：为什么做、做了什么、为什么下线、关键代码引用
- 避免彻底遗忘导致未来重复踩坑
- 为重新引入提供前置条件清单

## 归档格式

每个废弃特性一个文件，命名：`<特性名>.md`

```markdown
# <特性名>

## 特性目标
当时想解决什么问题

## 实现摘要
做了什么，关键改动

## 下线原因
为什么决定移除或暂停

## 关键引用
- 相关 git commit hash
- 关键文件路径
- 相关 SPARK/EXP 编号

## 重新引入的前置条件
满足什么条件后可以考虑捡回来
```
